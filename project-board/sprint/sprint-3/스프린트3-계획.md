## 3. 구현 전략

### 3-1. 다크 모드 방식

- **제어 속성**: `<html data-theme="dark">` — CSS `[data-theme="dark"]` 셀렉터로 모든 오버라이드 적용
- **초기화 순서**: `localStorage.getItem('theme')` → 없으면 `prefers-color-scheme` 자동 감지
- **FOUC 방지**: `<head>` 안 인라인 `<script>`로 DOM 렌더 전 테마 적용
- **localStorage 키**: `theme` (`'dark'` | `'light'`)
- **토글 버튼**: 각 페이지 헤더/nav에 🌙/☀️ 버튼 추가 (`addEventListener` 바인딩)

**CSS 변수 패턴** (전 파일 공통):

```css
:root {
  --bg-page: #f8fafc;  --bg-card: #ffffff;
  --text-main: #1e293b;  --text-sub: #475569;
  --border: #e2e8f0;  --input-bg: #ffffff;
}
[data-theme="dark"] {
  --bg-page: #0f172a;  --bg-card: #1e293b;
  --text-main: #f1f5f9;  --text-sub: #94a3b8;
  --border: #334155;  --input-bg: #0f172a;
}
```

### 3-2. workflow.html Phase 색상 (다크 모드 전용)

Phase 별 배경색은 JS에서 `style.setProperty`로 주입하므로 별도 `PHASE_DARK_BG` 맵 추가:

```javascript
const PHASE_DARK_BG = {
  '#8b5cf6': '#2d1b52',  '#0ea5e9': '#0c2946',
  '#10b981': '#0c3325',  '#f59e0b': '#3d2a00',
  '#ec4899': '#3d0c27',  '#0f172a': '#1e2a3a',
};
```

Phase 요소 렌더 시 `--bg-dark` CSS 변수 주입 → `[data-theme="dark"] .phase { background: var(--bg-dark); }`

## 4. 실행 내역

### 4-1. `style/project-board.css` ✅

- `:root`에 시맨틱 변수 6종 추가 (`--bg-page`, `--bg-card`, `--text-main`, `--text-sub`, `--border`, `--input-bg`)
- `[data-theme="dark"]` 블록 추가
- 기존 하드코딩 색상(`#fff`, `#1e293b`, `#e2e8f0` 등) → `var()` 참조로 교체
  - `body`, `nav`, `.nav-link`, `.hero-badge`, `.btn-outline`
  - `.problem-card`, `.problem-text`, `.tool-card`, `.tool-btn-outline`
  - `.feat-card`, `.flow-body`, `.flow-step::after`
- `[data-theme="dark"] nav` — `rgba(15,23,42,.92)` 반투명 오버라이드
- `[data-theme="dark"] .hero` — 다크 그라디언트 오버라이드
- `.theme-btn`, `.section-gray`, `.section-white` 신규 클래스 추가
- `@media(max-width:400px)` 추가

### 4-2. `project-board.html` ✅

- `<head>` 최상단 인라인 `<script>` — 테마 init (FOUC 방지)
- nav `.nav-links`에 `<button class="theme-btn" id="theme-btn">` 추가
- `<section style="background:#f8fafc">` → `class="section-gray"` 교체 (인라인 색상 제거)
- `<section style="background:#fff">` → `class="section-white"` 교체
- `<script>` 블록에 `toggleTheme()`, `updateThemeBtn()`, `addEventListener` 추가

### 4-3. `docs/workflow.html` ✅

- `<head>` 최상단 인라인 `<script>` — 테마 init
- `<style>` 블록 전면 업데이트:
  - `:root` + `[data-theme="dark"]` 변수 추가
  - body, header, flow-item, phase, step, file-tag, checklist-item 등 전체 색상 → var() 교체
  - `[data-theme="dark"] .phase`, `.qc-head`, `.log-head`, `.git-head`, `.tip`, `.step.qa` 오버라이드
  - `@media(max-width:600px)` 신규 (wrap, phase-head, prompt-block)
  - `@media(max-width:400px)` 신규 (wrap, progress-wrap, copy-btn)
- `PHASE_DARK_BG` 상수 추가 (JS 최상단)
- `render()` — phase 요소에 `--bg-dark` CSS 변수 주입, 테마 감지해 open 시 bg 적용
- `togglePhase()` — dark mode 시 `PHASE_DARK_BG` 사용
- `applyThemeToPhaseHeads()` — 테마 전환 즉시 열린 phase 헤드 업데이트
- `.header`에 `<button class="theme-btn" id="theme-btn">` 추가
- `toggleTheme()`, `updateThemeBtn()`, `addEventListener` 추가
