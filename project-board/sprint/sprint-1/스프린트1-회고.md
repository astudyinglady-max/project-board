## KPT 회고

### Keep (계속할 것)

- **Phase 단위 QC 체크리스트** — 요구사항·유저스토리·기술 스택 검증 항목을 명시해 놓아, 완료 기준이 분명했음.
- **sprint-log.md 정기 기록** — 확정 요구사항, 기술 스택 결정 이유, 논의 사항, 다음 액션을 한 문서에 모아 두어 다음 Phase 착수 시 참고하기 좋았음.
- **이슈목록 + 워크플로우 Step 매핑** — [기획-01] ↔ p1s0 등 연결로, 이슈와 CLI 워크플로우가 1:1 대응되어 진행 추적이 수월했음.
- **제로 의존성·서버리스 원칙 문서화** — 기술스택.md·요구사항에서 “설치/계정/인터넷 불필요”를 명시해, 스코프 크리프 방지에 도움이 됨.

### Problem (문제점)

- **폴더구조.md 이월** — [기획-03] 수용 기준에 포함된 폴더 구조 문서를 Phase 1 내 완료하지 못하고 후속 작업으로 남김.
- **단일 sprint-log** — Phase 1만 로그가 있고, 디자인/프론트/백엔드 Phase별 로그가 없어 스프린트 전체 흐름을 한눈에 보기 어려움.
- **테스트는 E2E만 존재** — 단위 테스트·커버리지 수치는 없고, Playwright E2E만 있어 “기술 지표”에서 단위 테스트 항목을 채우기 어려움.

### Try (시도할 것)

- **Phase 2 착수 전 폴더구조.md 완료** — 다음 스프린트 초반에 폴더구조.md 작성/보완 후 디자인 Phase 진행.
- **Phase별 sprint-log 유지** — 디자인/프론트/백엔드 Phase 완료 시마다 해당 Phase용 sprint-log 섹션 또는 파일을 추가해 이력 축적.
- **CI 연동** — GitHub Actions CI 도입 시 E2E(Playwright)를 워크플로우에 포함해, PR 기준으로 자동 검증.
- **이슈 실제 등록** — 이슈목록.md 내용을 GitHub Issues로 옮겨 보드·라벨과 연동해 가시성 확보.

## 기술 부채 — 무엇부터 해결하면 좋을지

### 우선순위 (효과 큰 순)

| 순위 | 기술 부채 | 해결 시 기대 효과 | 소요·난이도 |
|------|----------|-------------------|------------|
| **1** | **GitHub Actions CI 미설정** | 코드 품질 점수 상승, PR 시 자동 검증, 회귀 방지. **한 번 설정하면 계속 이득.** | 1~2h, 낮음 |
| **2** | **CI에 E2E(Playwright) 미포함** | 테스트 점수·신뢰도 상승. 이미 E2E 42개 있으므로 워크플로우만 추가하면 됨. | 0.5~1h, 낮음 |
| **3** | **폴더구조.md 미보완** | 문서화 10점 만점, [기획-03] 이월 해소. | 0.5h, 낮음 |
| 4 | 단위 테스트/커버리지 없음 | 테스트 점수 추가 상승. 다만 정적 HTML/JS라 빌드·테스트 환경 도입 비용 대비 효과는 상대적으로 작음. | 2~4h, 중간 |
| 5 | file:// 안내 강화 | 사용자 혼란 감소. 점수 직접 영향은 작음. | 0.5h, 낮음 |

### 추천 실행 순서 (점수·품질 동시에 올리기)

1. **폴더구조.md** 보완 → 문서화 10점 + 이월 정리.
2. **`.github/workflows/ci.yml`** 추가  
   - HTML 유효성 검사(선택), **ESLint**, **Playwright E2E** 실행 → 코드 품질 + 테스트 점수 상승.
3. **`.github/workflows/deploy.yml`** 추가 (main → GitHub Pages) → 배포 자동화, 운영 안정성.
4. (여유 시) **ESLint 설정** (`eslint.config.js` 또는 `.eslintrc`) — CI가 의미 있으려면 규칙 정의 필요.
5. (선택) **핵심 유틸만 단위 테스트** — 예: `kit/cli.mjs`의 상태 읽기/쓰기 로직을 분리한 뒤 테스트. 정적 페이지 위주라 우선순위는 1~3보다 낮음.

**정리**: **CI(ESLint + E2E) + 폴더구조.md**까지 해결하면 78점 → **82~85점** 수준 기대. Phase 2·3 진행으로 기능·UX를 채우면 **88~95점**까지도 가능.
