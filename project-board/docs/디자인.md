## 2. 타이포그래피

### 2-1. 폰트 패밀리

```css
body {
  font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
}
```

외부 폰트(Google Fonts 등) **사용 금지** — 제로 의존성 원칙.

### 2-2. 사이즈 스케일

| 토큰 | 크기 | 용도 |
|------|------|------|
| `text-xs` | `11px` | 뱃지, 메타, 스텝 번호 |
| `text-sm` | `12px` | 라벨, 태그, 섹션 타이틀 |
| `text-base` | `13px` | 본문, 링크, 입력값 |
| `text-md` | `14px` | 일반 텍스트, 버튼 |
| `text-lg` | `15px` | 카드 제목, 페이즈 이름 |
| `text-xl` | `16px` | 로고, 도구 제목 |
| `text-2xl` | `18px` | 보드 제목, 헤더 h1 |
| `text-3xl` | `22px` | 워크플로우 헤더 h1 |
| `text-hero` | `clamp(28px, 5vw, 52px)` | 랜딩 히어로 제목 |

### 2-3. 폰트 웨이트

| 웨이트 | 사용 상황 |
|--------|----------|
| `400` (기본) | 일반 본문, 설명 텍스트 |
| `600` | 라벨, 탭, 링크, 강조 본문 |
| `700` | 카드 제목, 버튼, 섹션 제목 |
| `800` | 페이지 제목, 히어로, 통계 수치 |

### 2-4. 라인 하이트

| 상황 | 값 |
|------|----|
| 단일 행 (버튼, 라벨) | `1` ~ `1.2` |
| 짧은 본문 | `1.5` |
| 긴 본문, 코드 | `1.7` ~ `1.8` |
| 히어로 제목 | `1.15` |

### 2-5. 기타 텍스트 규칙

- **letter-spacing**: 섹션 레이블에만 `0.08em` 사용
- **text-transform**: 사용하지 않음
- **text-decoration**: 완료 항목에만 `line-through` 사용

## 3. 간격 시스템 (4px 단위)

모든 `padding`, `margin`, `gap`은 **4의 배수**를 사용합니다.

### 3-1. 간격 토큰

| 토큰 | 값 | 대표 사용처 |
|------|----|------------|
| `sp-1` | `4px` | 최소 간격, 탭 gap |
| `sp-2` | `8px` | 컴포넌트 내부 gap, 버튼 padding |
| `sp-3` | `12px` | 내부 padding, form gap |
| `sp-4` | `16px` | 카드 padding, 섹션 내부 padding |
| `sp-5` | `20px` | 카드 padding (큰), 섹션 간격 |
| `sp-6` | `24px` | 레이아웃 수평 padding |
| `sp-7` | `28px` | 섹션 하단 margin |
| `sp-8` | `32px` | 섹션 간격 |
| `sp-12` | `48px` | 버튼 그룹 margin-top, 큰 섹션 padding |
| `sp-15` | `60px` | 페이지 섹션 상하 padding |
| `sp-20` | `80px` | 히어로 상단 padding |

### 3-2. 레이아웃 구조 간격

```
페이지 수평 패딩: 24px (양쪽)
섹션 상하 패딩:  60px (큰 섹션), 48px (중간 섹션)
카드 내부 패딩:  20px (일반), 16px (콤팩트)
컴포넌트 gap:    16px (카드 그리드), 8px (인라인 요소)
```

### 3-3. Border Radius 규칙

| 크기 | 값 | 사용 대상 |
|------|----|----------|
| 원형 | `999px` | 뱃지, pill 버튼 |
| 큰 | `14–16px` | 메인 카드, 페이즈 블록 |
| 중간 | `12px` | 프로젝트 카드, 폼 카드 |
| 기본 | `8–10px` | 버튼, 입력, 스텝 항목 |
| 작은 | `5–6px` | 태그, 체크박스, 코드 블록 |
| 최소 | `3–4px` | 미니 바, 프리뷰 바 |

## 4. 컴포넌트 규칙

### 4-1. 버튼 (Button)

#### 종류

| 종류 | 클래스 | 용도 |
|------|--------|------|
| Primary | `.btn-primary` | 주요 액션 (저장, 생성, CTA) |
| Outline | `.btn-outline` | 보조 액션 |
| Green | `.btn-green` | 성공/시작 액션 |
| Danger | `.btn-danger` | 삭제 액션 |
| Text | `.btn-text` | 최소 강조 액션 |

#### 스타일 규칙

```css
/* 공통 */
.btn {
  padding: 8px 16px;        /* 기본 크기 */
  border-radius: 8px;
  font-size: 13–14px;
  font-weight: 700;
  cursor: pointer;
  border: none;
  transition: all .15s;
}

/* Primary */
.btn-primary {
  background: var(--indigo);
  color: #fff;
}
.btn-primary:hover {
  background: var(--indigo-d);
  transform: translateY(-1px);
}

/* Outline */
.btn-outline {
  background: #fff;
  border: 1.5px solid var(--gray-200);
  color: var(--gray-800);
}
.btn-outline:hover {
  border-color: var(--indigo);
  color: var(--indigo);
}

/* Danger */
.btn-danger {
  background: #fee2e2;
  color: #dc2626;
  border: 1px solid #fecaca;
}
```

#### 버튼 규칙

- 버튼 텍스트는 **현재형 동사**로 작성 (`저장`, `삭제`, `추가`)
- 아이콘 + 텍스트 조합 시 gap: `6px`
- disabled 상태: `opacity: 0.5; cursor: not-allowed`
- 로딩 상태: 텍스트 변경 또는 spinner (CSS only)

### 4-2. 입력 (Input / Select / Textarea)

#### 스타일 규칙

```css
input, select, textarea {
  padding: 8px 12px;
  border: 1.5px solid var(--gray-200);
  border-radius: 8px;
  font-size: 13px;
  color: var(--gray-800);
  background: #fff;
  transition: border-color .15s;
  width: 100%;
  box-sizing: border-box;
}

input:focus, select:focus, textarea:focus {
  outline: none;
  border-color: var(--indigo);
  box-shadow: 0 0 0 3px var(--indigo-l);
}

input::placeholder {
  color: var(--gray-400);
}

textarea {
  resize: vertical;
  min-height: 80px;
  line-height: 1.6;
}
```

#### 라벨 규칙

```css
label {
  display: block;
  font-size: 12px;
  font-weight: 600;
  color: var(--gray-600);
  margin-bottom: 4px;
}
```

#### 입력 규칙

- `<label>`은 항상 `<input>`과 짝지어 사용
- 에러 상태: `border-color: #dc2626; background: #fff5f5`
- 에러 메시지: `font-size: 11px; color: #dc2626; margin-top: 4px`
- 필드 그룹 gap: `12px` (form-grid)
- `type="number"` 입력: `-moz-appearance: textfield; appearance: none` (화살표 숨김)

### 4-3. 카드 (Card)

#### 종류

| 종류 | 클래스 | 용도 |
|------|--------|------|
| 프로젝트 카드 | `.project-card` | 프로젝트 보드 항목 |
| 폼 카드 | `.form-card` | 입력 폼 래퍼 |
| 도구 카드 | `.tool-card` | 랜딩 기능 소개 |
| 피쳐 카드 | `.feat-card` | 특징 소개 |
| 페이즈 블록 | `.phase-block` | 워크플로우 단계 |

#### 스타일 규칙

```css
/* 기본 카드 */
.card {
  background: #fff;
  border: 1.5px solid var(--gray-200);
  border-radius: 12px;
  padding: 16–20px;
}

/* Hover 효과 (인터랙티브 카드) */
.card:hover {
  border-color: var(--indigo);
  box-shadow: 0 4px 12px rgba(99, 102, 241, .1);
  transform: translateY(-2px);    /* 선택적 */
  transition: all .2s;
}

/* 페이즈 블록 (좌측 컬러 바) */
.phase-block {
  border-left: 4px solid [phase-color];
  border-radius: 14px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, .07);
}
```

#### 카드 규칙

- 카드 내부 구조: 헤더 / 본문 / 액션 순서
- 중첩 카드는 최대 **2단계** (카드 안의 카드)
- 카드 그리드 gap: `16px`
- 반응형: 모바일에서 1컬럼으로 전환

### 4-4. 모달 (Modal)

#### 스타일 규칙

```css
/* 오버레이 */
.modal-overlay {
  position: fixed;
  inset: 0;
  background: rgba(15, 23, 42, .5);   /* --gray-900 + opacity */
  display: flex;
  align-items: center;
  justify-content: center;
  z-index: 1000;
  padding: 20px;
}

/* 모달 본체 */
.modal {
  background: #fff;
  border-radius: 16px;
  padding: 24px;
  width: 100%;
  max-width: 480px;
  max-height: 90vh;
  overflow-y: auto;
  box-shadow: 0 20px 60px rgba(0, 0, 0, .15);
}

/* 모달 헤더 */
.modal-header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  margin-bottom: 20px;
}

.modal-title {
  font-size: 16px;
  font-weight: 700;
}

/* 닫기 버튼 */
.modal-close {
  width: 28px;
  height: 28px;
  border-radius: 6px;
  border: none;
  background: var(--gray-100);
  cursor: pointer;
  font-size: 16px;
  color: var(--gray-600);
}
.modal-close:hover {
  background: var(--gray-200);
}

/* 모달 푸터 */
.modal-footer {
  display: flex;
  justify-content: flex-end;
  gap: 8px;
  margin-top: 20px;
  padding-top: 16px;
  border-top: 1px solid var(--gray-200);
}
```

#### 모달 규칙

- `ESC` 키로 닫기 — 반드시 구현
- 배경(overlay) 클릭으로 닫기 — 반드시 구현
- 모달 열릴 때 `<body>` overflow: hidden 처리
- 포커스 트랩: 모달 내 `Tab` 키가 모달 밖으로 빠져나가지 않도록
- 확인/취소 버튼 순서: 취소(outline) → 확인(primary)
- 삭제 확인 모달: 확인 버튼은 `.btn-danger` 사용

## 7. 금지 사항

### 7-1. 컬러

| 금지 | 이유 | 대안 |
|------|------|------|
| 임의 hex 색상 직접 사용 | 일관성 깨짐 | CSS 변수 사용 |
| `rgba(0,0,0,1)` 순수 검정 텍스트 | 눈에 너무 강함 | `var(--gray-800)` |
| 대비율 3:1 미만 색상 조합 | 접근성 기준 미달 | WCAG AA 기준(4.5:1) 충족 |
| CSS 변수 없이 색상 반복 선언 | 유지보수 어려움 | `:root`에 한 번만 선언 |

### 7-2. 타이포그래피

| 금지 | 이유 | 대안 |
|------|------|------|
| 외부 폰트 import (Google Fonts 등) | 제로 의존성 원칙 위반, 오프라인 불가 | 시스템 폰트 스택 |
| 임의 폰트 크기 (예: `17px`, `19px`) | 스케일 일관성 깨짐 | 정의된 스케일 사용 |
| `font-weight: 500` | 시스템 폰트에서 지원 불안정 | `400` 또는 `600` 사용 |
| `text-transform: uppercase` 남용 | 가독성 저하 | 섹션 레이블에만 제한적 사용 |

### 7-3. 간격

| 금지 | 이유 | 대안 |
|------|------|------|
| 4의 배수가 아닌 임의 값 (예: `5px`, `7px`, `15px`) | 그리드 일관성 깨짐 | 4px 단위 값 사용 |
| `px` 단위가 아닌 `em`/`rem` 간격 | 시스템 불일치 | `px` 사용 (예외: font-size의 line-height) |
| 인라인 `style=""` 속성으로 간격 지정 | 유지보수 불가 | CSS 클래스 사용 |

### 7-4. 컴포넌트

| 금지 | 이유 | 대안 |
|------|------|------|
| `onclick=""` 인라인 이벤트 | 코딩 컨벤션 위반 | `addEventListener` |
| 모달에서 ESC/배경 닫기 미구현 | UX 기준 미달 | 반드시 두 방식 모두 구현 |
| 버튼 없는 폼 제출 (`Enter` 만) | 접근성 부족 | 명시적 submit 버튼 포함 |
| 카드 중첩 3단계 이상 | 복잡도 과증가 | 최대 2단계 |
| `z-index` 임의 값 | 레이어 충돌 | 모달: 1000, 네비게이션: 100 |

### 7-5. 일반

| 금지 | 이유 |
|------|------|
| CSS 프레임워크 (Tailwind, Bootstrap 등) CDN 추가 | 제로 의존성 원칙 위반 |
| `!important` 남용 | 우선순위 관리 불가 |
| 애니메이션 `transition` 0.5s 초과 | 사용자 반응성 저하 |
| 고정 높이(`height: Npx`)로 텍스트 컨테이너 설정 | 텍스트 오버플로우 위험 |
