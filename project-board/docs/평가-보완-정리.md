**작성일**: 2026-03-14

## 1. CI/CD 및 자동화 보완

### 문제
- `.github/workflows/`에 ci.yml, deploy.yml이 **없다**고 평가됨 (실제로는 **저장소 루트**에 존재).
- ESLint는 있으나 CI 파이프라인에 연결되지 않은 것으로 보임.
- GitHub Pages만 수동 배포 가능한 상태로 인식됨.

### 보완 내용

| 항목 | 조치 |
|------|------|
| **워크플로우 위치** | 저장소 루트 `.github/workflows/` (project-board/ 폴더 밖). README에 명시. |
| **ci.yml** | `defaults.run.working-directory: project-board` 추가 → package.json이 project-board/ 안에 있어도 CI 정상 실행. 트리거: PR/푸시 시 `master`, `main`, `develop`. |
| **deploy.yml** | master/main 푸시 시 GitHub Pages 자동 배포 (path: project-board). |
| **ESLint** | CI의 lint job에서 `npx eslint . --max-warnings 0` 실행 (연동 완료). |
| **문서화** | README에 「CI/CD 및 자동화」 섹션 추가 — 워크플로우 위치, 트리거, Pages 설정 안내. |

### 확인 방법
- GitHub → Actions에서 **CI** (PR/푸시 시), **Deploy to GitHub Pages** (master/main 푸시 시) 실행 여부 확인.
- Settings → Pages → Source를 **GitHub Actions**로 설정했는지 확인.

## 2. 테스트 전략 보완

### 문제
- E2E 42개는 있으나 단위 테스트 없음.
- E2E도 구현 미흡으로 실패 가능성 높다고 평가됨.
- 테스트 커버리지 수치 없음.

### 현재 상태 (문서화)

| 구분 | 내용 |
|------|------|
| **E2E** | Playwright, `tests/e2e/` — landing 6 + workflow 19 + project-board 17 = **42개**. 모바일 375px 포함. |
| **실행** | `npm test` (로컬 서버 자동 기동). CI에서 PR/푸시 시 자동 실행. |
| **단위 테스트** | 없음. 정적 HTML/JS·빌드 없음 구조라 선택 도입 예정. |
| **커버리지** | E2E는 시나리오 커버리지(핵심 플로우). 수치 리포트 미도입. |

README에 **「테스트 전략」** 섹션 추가해 위 내용 명시.

### 추가 보완 제안 (선택)
- **E2E 실패 줄이기**: `npm test` 로컬 실행 후 실패 스펙만 수정 (선택자·대기 시간 등).
- **단위 테스트**: `kit/cli.mjs` 등 핵심 로직만 작은 스크립트로 분리 후 테스트.
- **커버리지**: “E2E 42개로 랜딩·워크플로우·보드 핵심 플로우 커버”를 문서로 명시.

## 5. 기술 스택 보완 (6/8)

### 문제
- package.json에 devDependencies(Playwright, ESLint)가 있어 "UI 제로 의존성"과 모순으로 보임. 설명 부재.

### 보완 내용
- **기술스택.md**에 **「제로 의존성 원칙과 devDependencies」** 섹션 추가.
  - **UI/배포물**: 의존성 0 — HTML/CSS/JS는 외부 라이브러리·빌드 없이 동작.
  - **CLI 실행**: 의존성 0 — Node 내장 모듈만 사용.
  - **devDependencies**: Playwright·ESLint는 **개발·CI 전용**, 배포 artifact에 미포함, 사용자 실행과 무관.
  - "런타임은 제로 의존성, 개발 환경에서만 품질 검증 도구 사용"으로 역할 분리 명시 → 모순 아님.
