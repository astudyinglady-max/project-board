## 폴더 역할 요약

| 폴더/파일 | 역할 | 수정 권한 |
|-----------|------|----------|
| `docs/` | GitHub Pages로 서빙되는 HTML + 기획 문서 | 자유롭게 수정 |
| `style/` | 외부 CSS (HTML별 스타일시트) | 자유롭게 수정 |
| `tests/` | Playwright E2E 테스트 스펙 | 자유롭게 수정 |
| `kit/` | 로컬 CLI (프로젝트 맞춤 수정 가능) | 자유롭게 수정 |
| `.kit/` | 업스트림 git submodule | **수정 금지** |
| `.workflow/config.json` | 워크플로우 단계 정의 | 수동 편집 가능 |
| `.workflow/state.json` | 현재 진행 상태 | CLI 명령어로만 수정 |
| `.workflow/state.js` | HTML용 상태 브릿지 | **자동 생성, 수정 금지** |
| `.workflow/status.md` | 사람이 읽는 상태 요약 | **자동 생성, 수정 금지** |
| `.workflow/templates/` | Step별 문서 초안 템플릿 | 자유롭게 수정 |
| `.github/` | GitHub 이슈·PR 템플릿 | 자유롭게 수정 |
| `agents/` | Claude Code 에이전트 정의 | 자유롭게 수정 |
| `skills/` | Claude Code 슬래시 커맨드 | 자유롭게 수정 |
| `sprint/` | 스프린트별 로그 및 이슈 | 자유롭게 수정 |

## 구현 현황 (Phase별)

| Phase | 산출물 | 상태 | 비고 |
|-------|--------|------|------|
| Phase 1 (기획) | 요구사항, 유저스토리, 기술스택, 차별화, 이슈목록 등 | 완료 | 문서 정의 완료 |
| Phase 2 (디자인) | 디자인.md, 컴포넌트목록.md, 페이지목록.md, 사용자흐름.md | 완료 | 디자인 문서 확정 |
| Phase 3~4 (구현) | project-board.html, docs/index.html, docs/workflow.html | 일부 구현 | 3개 HTML 존재. 칸반·워크플로우·랜딩 핵심 기능 동작. 완성도는 스프린트 진행에 따라 확대 예정 |

3개 HTML 파일은 Must Have 범위의 핵심(워크플로우 가이드, 프로젝트 보드, 랜딩)을 담고 있으며, Phase 2 디자인 산출물을 바탕으로 스프린트 단위로 기능을 보강하는 구조입니다.
