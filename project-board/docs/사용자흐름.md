## 시나리오 1. 신규 사용자 온보딩

**목표**: 툴킷을 처음 접한 사용자가 가치를 이해하고, 워크플로우 또는 보드로 진입하는 흐름.

| 단계 | 사용자 행동 | 페이지/위치 | 결과 |
|------|------------|------------|------|
| 1 | 랜딩 페이지 접속 | project-board.html (또는 GitHub Pages 루트) | 히어로에서 가치 제안·통계 확인 |
| 2 | 스크롤 또는 네비 클릭으로 "What's included" 확인 | #features | workflow / board / landing 3개 도구 카드 노출 |
| 3 | "How it works"에서 사용 방법 3단계 확인 | #how | ① 설정 → ② 워크플로우 → ③ 보드 관리 흐름 인지 |
| 4 | "체험하기" 또는 "워크플로우" 카드 클릭 | 랜딩 | docs/workflow.html로 이동 |
| 5 | (대안) "프로젝트 보드" 카드 클릭 | 랜딩 | docs/index.html로 이동 |
| 6 | (선택) "Install" 섹션에서 복사 버튼으로 설치 명령 복사 | #install | 로컬 클론·실행 방법 확보 |
| 7 | (선택) 우하단 플로팅 "체험하기" 클릭 | 랜딩 | 보드 또는 워크플로우로 이동 |

**완료 기준**: 사용자가 랜딩에서 **워크플로우** 또는 **보드** 중 하나로 진입한 상태.

## 시나리오 2. 워크플로우 진행

**목표**: Phase 0~5 단계를 따라 Step 완료 → 프롬프트 복사 → LOG 기록 → QC 체크 → 다음 단계로 진행하는 흐름.

| 단계 | 사용자 행동 | 페이지/위치 | 결과 |
|------|------------|------------|------|
| 1 | 워크플로우 가이드 접속 | docs/workflow.html | 6개 Phase, 전체 진도 0% 표시 |
| 2 | Phase 0(또는 현재 Phase) 헤더 클릭 | Phase 블록 | 해당 Phase 본문 펼쳐짐 |
| 3 | Step 헤더 클릭 | Step 영역 | 해당 Step의 "생성할 파일", "프롬프트 예시" 노출 |
| 4 | "복사" 버튼 클릭 | 프롬프트 블록 | 클립보드에 프롬프트 복사, 버튼이 "✓ 복사됨"으로 변경 |
| 5 | Claude Code 등에서 프롬프트 실행 후 산출물 작성 | (외부) | Step 완료 |
| 6 | (해당 Phase에 QC 있는 경우) "QC" 섹션 헤더 클릭 | QC 섹션 | 체크리스트 펼쳐짐 |
| 7 | QC 체크리스트 항목 클릭 | 체크리스트 | 항목 체크/해제, 전체 진도율 자동 갱신 |
| 8 | (선택) "전체 초기화" 클릭 후 확인 | 상단 버튼 | localStorage 초기화, 진도 0%로 복귀 |
| 9 | (선택) "파일로 저장" 클릭 | 상단 버튼 | 진행 상태 JSON 파일 다운로드 |
| 10 | 새로고침 | — | localStorage에 저장된 열린 Phase·QC 체크 상태 유지 |

**완료 기준**: 한 Phase 내에서 Step 열기 → 프롬프트 복사 → QC 체크까지 수행하고, 새로고침 후에도 상태가 유지됨.

## 시나리오 3. 프로젝트 보드 관리

**목표**: 멀티 프로젝트 생성·선택·Phase별 Step 체크·진행률 확인·내보내기/삭제까지 수행하는 흐름.

| 단계 | 사용자 행동 | 페이지/위치 | 결과 |
|------|------------|------------|------|
| 1 | 프로젝트 보드 접속 | docs/index.html | "프로젝트 목록" 탭, 빈 상태 또는 기존 프로젝트 카드 표시 |
| 2 | "➕ 새 프로젝트 추가" 폼에 프로젝트명·설명 등 입력 후 "프로젝트 추가" 클릭 | 프로젝트 목록 탭 | 프로젝트 생성, 자동으로 "작업 보드" 탭 전환 |
| 3 | 작업 보드에서 Phase 블록 헤더 클릭 | 보드 본문 | 해당 Phase의 Step 목록 펼침/접힘 |
| 4 | Step 왼쪽 체크박스(원) 클릭 | Step 항목 | Step 완료/미완료 토글, 전체 진도 %·완료 수 갱신 |
| 5 | "프로젝트 목록" 탭 클릭 | 탭 | 프로젝트 카드 목록으로 복귀 |
| 6 | 다른 프로젝트 카드 클릭 | 프로젝트 카드 | 해당 프로젝트 선택, 보드 탭에서 해당 프로젝트 Phase/Step 표시 |
| 7 | "보드 열기" 버튼 클릭 | 프로젝트 카드 | 해당 프로젝트 보드로 전환 |
| 8 | "내보내기" 클릭 | 헤더 | 전체 보드 상태 JSON 파일 다운로드 |
| 9 | (선택) "삭제" 클릭 후 확인 | 프로젝트 카드 | 해당 프로젝트 삭제, 목록에서 제거 |
| 10 | 새로고침 | — | localStorage에 저장된 프로젝트·Step 체크 상태 유지 |

**완료 기준**: 프로젝트 1개 이상 생성 → 보드에서 Step 체크 → 진도율 확인 → (선택) 내보내기/삭제 후에도 다른 프로젝트 데이터가 유지됨.

## 페이지 간 이동 요약

```
[ 랜딩 project-board.html ]
        │
        ├── "워크플로우" / "체험하기" ──→ [ docs/workflow.html ]
        │
        └── "프로젝트 보드" ──────────→ [ docs/index.html ]

[ docs/workflow.html ]  ←→  (같은 탭에서 직접 URL 이동 시)  ←→  [ docs/index.html ]
```

- 랜딩에서 두 도구(워크플로우 / 보드)로 진입 가능.
- 워크플로우 ↔ 보드는 브라우저 주소창 또는 랜딩 네비로 이동.
