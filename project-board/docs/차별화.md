# 차별화 포인트

**프로젝트**: claude-commin-kit (AI 네이티브 개발 워크플로우 툴킷)
**작성일**: 2026-03-13

---

## 1. 해결하려는 문제의 실재성

### 문제 정의

> AI(Claude)와 함께 개발하는 솔로 개발자가, 기획~배포의 반복적인 스프린트 사이클을 진행할 때, **"지금 뭘 해야 하는지"와 "어디까지 했는지"를 매번 직접 추적해야 한다**는 인지 부하 문제.

AI와 대화형으로 개발하는 방식은 빠르고 강력하지만, 대화 세션은 무상태(stateless)다. 새 대화를 시작하면 이전 컨텍스트가 사라지고, 개발자는 다시 상황을 설명하며 "지금 Phase 2 디자인 단계야, 컴포넌트 목록 작성해줘"를 처음부터 타이핑해야 한다. 워크플로우의 어느 단계인지, 무엇이 완료됐는지, 다음은 무엇인지를 개발자 본인이 머릿속에서 관리한다.

### 문제의 크기

- **대상 사용자**: AI 코딩 어시스턴트(Claude, GPT, Cursor 등)를 사용해 혼자 프로젝트를 진행하는 개발자 — 1인 스타트업, 사이드 프로젝트 개발자, 프리랜서
- **발생 빈도**: 매 스프린트, 매 대화 세션마다. AI와 함께 개발하면 하루에도 수십 번 새 대화를 시작한다
- **현재 해결 방식**: 노션/옵시디언에 수동으로 체크리스트 관리, 깃허브 이슈 직접 작성, 머릿속에서 상태 추적, 혹은 관리 자체를 포기
- **해결 방식의 한계**: 워크플로우 관리 도구와 AI 작업 도구가 분리되어 있어 컨텍스트 전환 비용이 크고, 프롬프트를 매번 새로 작성해야 한다

### 근거 데이터

- Claude, ChatGPT 등 AI 코딩 도구 사용자 커뮤니티(Reddit r/LocalLLaMA, X/Twitter 개발자 스레드)에서 반복적으로 등장하는 불만: *"AI가 이전 대화를 기억하지 못해서 매번 설명을 다시 해야 한다"*
- AI 코딩 도구 사용자의 가장 큰 생산성 병목은 **컨텍스트 유지**와 **반복 프롬프트 작성**임이 다수 사례에서 확인됨

## 2. 기존 솔루션의 한계

| 솔루션 | 장점 | 한계 |
|--------|------|------|
| Jira / Linear | 강력한 이슈 트래킹, 팀 협업 | 설정 복잡, 무겁고 유료, AI 워크플로우 개념 없음, 서버 필요 |
| Notion | 유연한 문서화, 템플릿 | AI 프롬프트 관리 불가, 오프라인 미지원, 외부 서비스 의존 |
| GitHub Issues | 코드와 통합, 무료 | 스프린트 단계 개념 없음, 프롬프트 관리 불가 |
| Cursor/Windsurf Rules | AI 컨텍스트 유지 가능 | 프로젝트 보드 기능 없음, IDE 종속, 워크플로우 가이드 없음 |
| 수동 체크리스트 (Markdown) | 단순, 도구 불필요 | 상태 자동화 없음, 브라우저 UI 없음, CLI 연동 없음 |

### 공통적인 한계

1. **AI 워크플로우에 특화되지 않음** — 기존 도구들은 "사람이 직접 코딩"하는 워크플로우를 전제로 설계됐다. AI에게 넘길 프롬프트를 관리하거나, AI 작업 결과물의 QC 체크리스트를 단계별로 제공하는 도구가 없다
2. **브라우저 외부 서비스 의존** — Jira, Notion, Linear는 모두 서버와 계정이 필요하다. 인터넷 없이 로컬에서 바로 쓸 수 없고, 서비스가 중단되거나 가격이 오르면 데이터 접근이 위협받는다
3. **CLI와 UI의 단절** — 터미널 작업(git, 파일 생성, 상태 확인)과 브라우저 UI 작업이 분리되어 있어 상태를 양쪽에서 별도로 관리해야 한다

## 3. 우리 서비스만의 강점

### 핵심 차별화 포인트

#### 차별화 1: AI 네이티브 워크플로우 — 프롬프트가 곧 단계다

> Claude에게 바로 붙여넣을 수 있는 단계별 프롬프트를 워크플로우에 내장

기존 솔루션에서는 "Phase 2 디자인 단계에서 컴포넌트 목록을 만들어야 한다"는 사실을 개발자가 알고, 프롬프트를 직접 작성해야 한다. 우리는 각 Step에 **검증된 프롬프트 템플릿**을 내장하여, 클릭 한 번으로 복사 후 Claude에 붙여넣으면 된다. 워크플로우 관리와 AI 작업 지시가 하나의 도구에서 이루어진다.

#### 차별화 2: 제로 의존성 — 설치도, 계정도, 인터넷도 불필요

> HTML 파일 하나를 브라우저로 열면 즉시 작동하는 서버리스 툴킷

외부 라이브러리, 번들러, 백엔드 서버, 사용자 계정이 없다. `git clone` 후 `index.html`을 더블클릭하면 전체 기능이 작동한다. 데이터는 localStorage에 저장되고 JSON으로 백업된다. Jira 계정이 없어도, Notion 요금제가 없어도, 인터넷 연결이 없어도 쓸 수 있다.

#### 차별화 3: CLI ↔ 브라우저 양방향 상태 동기화

> 터미널에서 `complete` 하면 브라우저 UI에 즉시 반영, 반대도 동일

`kit/cli.mjs`로 터미널에서 상태를 바꾸면 `state.js`가 재생성되고, 브라우저를 새로고침하면 변경사항이 반영된다. 코딩은 터미널에서, 진행 현황은 브라우저에서 — 양쪽이 항상 같은 진실(Single Source of Truth)을 바라본다.

#### 차별화 4: 6단계 QC 내장 — "완성"의 정의가 도구 안에 있다

> 각 Phase 완료 전 품질 검증 체크리스트를 강제하는 가드레일

AI와 함께 빠르게 코드를 생성하다 보면 QA를 건너뛰기 쉽다. 우리 툴킷은 Phase 완료 전 **QC 체크리스트**를 UI와 CLI 양쪽에서 제공하여, "완성됐다"는 기준을 도구 수준에서 강제한다. 개발자가 기억에 의존하지 않아도 된다.

## 포지셔닝

**claude-commin-kit은**
AI(Claude)와 함께 스프린트 단위로 개발하는 솔로 개발자를 위한 브라우저 전용 워크플로우 툴킷으로,
기존 프로젝트 관리 도구(Jira, Notion)와 달리 **AI 프롬프트 관리 + 제로 의존성 + CLI 동기화**를 하나로 통합하여 제공합니다.

## 구현 현황 · 검증 계획

| 차별화 포인트 | 구현 상태 | 비고 |
|---------------|:---------:|------|
| ① AI 네이티브 워크플로우(프롬프트 내장) | ✅ | docs/workflow.html — Phase별 프롬프트 원클릭 복사 |
