# 접근성 체크리스트 (WCAG AA)

**프로젝트**: claude-commin-kit (AI 네이티브 개발 워크플로우 툴킷)
**기준**: WCAG 2.1 Level AA
**적용 범위**: project-board.html, docs/index.html, docs/workflow.html

> 비기능 요구사항(요구사항.md)의 **접근성: WCAG AA 기준 (4.5:1 컬러 대비 충족)** 을 검증하기 위한 체크리스트입니다.  
> 개발·QA 시 항목별로 확인 후 체크합니다.

---

## 1. 인지 가능 (Perceivable)

### 1.1 텍스트·컬러 대비

| # | 체크 항목 | WCAG 기준 | 검증 방법 | 상태 |
|---|----------|-----------|----------|:----:|
| 1.1.1 | 일반 텍스트와 배경의 대비율 | 4.5:1 이상 (AA) | [Contrast Checker](https://webaim.org/resources/contrastchecker/) 또는 DevTools로 텍스트/배경 색상 입력 후 비율 확인 | ☐ |
| 1.1.2 | 큰 텍스트(18px 이상 또는 14px bold)와 배경 | 3:1 이상 (AA) | 동일 도구로 큰 제목·강조 문구 확인 | ☐ |
| 1.1.3 | UI 컴포넌트·그래픽과 인접 색상 | 3:1 이상 (AA, 1.4.11) | 버튼·입력창·테두리와 배경 조합 확인 | ☐ |

**프로젝트 적용**: `docs/디자인.md`의 컬러 시스템(CSS 변수) 사용. `--gray-800`(#1e293b) on `--gray-50`(#f8fafc) 등 조합이 4.5:1 이상인지 페이지별로 확인.

### 1.2 대체 텍스트

| # | 체크 항목 | WCAG 기준 | 검증 방법 | 상태 |
|---|----------|-----------|----------|:----:|
| 1.2.1 | 의미 있는 이미지에 `alt` 제공 | 1.1.1 | `<img>`에 적절한 alt 또는 `alt=""`(장식용) | ☐ |
| 1.2.2 | 아이콘만 있는 버튼/링크에 접근 가능한 이름 | 2.1.1, 4.1.2 | `aria-label`, `title` 또는 보이지 않는 텍스트 | ☐ |

**프로젝트 적용**: 복사 버튼, Phase 접기/펼치기 버튼, 프로젝트 보드의 추가/삭제 등 아이콘 버튼에 `aria-label` 또는 유의미한 텍스트 존재 여부 확인.

### 1.3 정보 구조

| # | 체크 항목 | WCAG 기준 | 검증 방법 | 상태 |
|---|----------|-----------|----------|:----:|
| 1.3.1 | 제목 계층 (`h1` → `h2` → …) 논리적 사용 | 1.3.1 | 한 페이지에 `h1` 하나, 순서 건너뛰기 없음 | ☐ |
| 1.3.2 | 폼 입력에 `<label>` 또는 `aria-label` 연결 | 1.3.1, 3.3.2 | 입력 필드와 레이블 연결 확인 | ☐ |

## 2. 조작 가능 (Operable)

### 2.1 키보드

| # | 체크 항목 | WCAG 기준 | 검증 방법 | 상태 |
|---|----------|-----------|----------|:----:|
| 2.1.1 | 모든 기능이 키보드만으로 조작 가능 | 2.1.1 | 마우스 없이 Tab, Enter, Space, Esc로 네비·버튼·아코디언·모달 동작 확인 | ☐ |
| 2.1.2 | 키보드 포커스가 콘텐츠에 갇히지 않음 | 2.1.2 | 모달·레이어 열었을 때 포커스 트랩 후 닫기 시 이전 위치로 복귀 | ☐ |

### 2.2 포커스

| # | 체크 항목 | WCAG 기준 | 검증 방법 | 상태 |
|---|----------|-----------|----------|:----:|
| 2.2.1 | 포커스 순서가 시각적 읽기 순서와 일치 | 2.4.3 | Tab 이동 시 논리적 순서(위→아래, 왼→오) 확인 | ☐ |
| 2.2.2 | 포커스 시각 표시가 명확함 (outline 또는 배경 변경) | 2.4.7 | `:focus` / `:focus-visible` 스타일이 3:1 이상 대비로 보임 | ☐ |

**프로젝트 적용**: `docs/디자인.md` 7-2. 컴포넌트 — 모든 인터랙티브 요소에 hover, focus 상태 정의 여부 확인.

### 2.3 터치·타겟 크기

| # | 체크 항목 | WCAG 기준 | 검증 방법 | 상태 |
|---|----------|-----------|----------|:----:|
| 2.3.1 | 터치/클릭 타겟 최소 44×44 CSS 픽셀 | 2.5.5 (AAA, 권장) | 유저스토리 수용 기준 반영 — 버튼·링크·체크박스 터치 영역 확인 | ☐ |
