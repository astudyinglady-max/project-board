## 유저스토리 목록

### Epic 1: 워크플로우 진행 가이드

#### US-01
**As a** 솔로 개발자,
**I want to** 6단계 Phase로 구성된 개발 워크플로우 가이드를 브라우저에서 확인하고 싶다,
**So that** 기획부터 배포까지 어떤 순서로 무엇을 해야 할지 놓치지 않을 수 있다.

**Acceptance Criteria:**
- [ ] Phase 0~5 전체 단계가 사이드바 또는 탭으로 표시된다
- [ ] 각 Phase 내 Step 목록이 순서대로 나열된다
- [ ] 현재 진행 중인 Phase/Step이 시각적으로 강조된다
- [ ] 완료된 Step은 완료 상태로 표시된다

**우선순위**: Must Have
**예상 포인트**: 3

---

#### US-02
**As a** 솔로 개발자,
**I want to** 각 Step의 프롬프트를 버튼 한 번으로 클립보드에 복사하고 싶다,
**So that** Claude에게 붙여넣기만 하면 바로 작업을 시작할 수 있다.

**Acceptance Criteria:**
- [ ] 각 Step에 "복사" 버튼이 존재한다
- [ ] 버튼 클릭 시 해당 프롬프트 전체가 클립보드에 복사된다
- [ ] 복사 성공 후 버튼 텍스트 또는 아이콘이 피드백 상태로 변경된다 (예: "복사됨 ✓")
- [ ] Clipboard API를 지원하지 않는 환경에서도 텍스트 선택 폴백이 동작한다

**우선순위**: Must Have
**예상 포인트**: 2

---

#### US-03
**As a** 솔로 개발자,
**I want to** Phase 완료 전 QC 체크리스트를 확인하고 싶다,
**So that** 각 단계의 산출물을 빠짐없이 검토하고 다음 단계로 넘어갈 수 있다.

**Acceptance Criteria:**
- [ ] 각 Phase에 QC 체크리스트 항목이 표시된다
- [ ] 체크리스트 항목을 개별 체크/언체크할 수 있다
- [ ] 모든 항목 체크 전 Phase 완료 처리 시 경고 또는 확인 다이얼로그가 표시된다
- [ ] 체크 상태는 localStorage에 저장되어 새로고침 후에도 유지된다

**우선순위**: Must Have
**예상 포인트**: 3

---

#### US-04
**As a** 솔로 개발자,
**I want to** 브라우저를 닫았다 다시 열어도 워크플로우 진행 상태가 복원되길 원한다,
**So that** 작업을 중단하고 나중에 돌아와도 어디까지 했는지 다시 찾을 필요가 없다.

**Acceptance Criteria:**
- [ ] Step 완료 상태, QC 체크 상태가 localStorage에 저장된다
- [ ] 페이지 새로고침 후 저장된 상태가 자동 복원된다
- [ ] 상태 저장 실패 시 (localStorage 용량 초과 등) 콘솔에 에러가 기록된다

**우선순위**: Must Have
**예상 포인트**: 2

---

#### US-05
**As a** CLI 사용자,
**I want to** 터미널에서 `node kit/cli.mjs sync` 명령어를 실행하면 브라우저 워크플로우 화면에 최신 상태가 자동 반영되길 원한다,
**So that** CLI와 브라우저 UI를 병행해도 상태가 항상 일치한다.

**Acceptance Criteria:**
- [ ] `sync` 명령어 실행 시 `state.js` 파일이 생성 또는 갱신된다
- [ ] `workflow.html`이 `state.js`를 로드하여 CLI 상태를 반영한다
- [ ] CLI에서 완료 처리한 Step이 브라우저 화면에서도 완료로 표시된다
- [ ] `state.js`가 없는 경우 브라우저 UI는 에러 없이 기본 상태로 동작한다

**우선순위**: Must Have
**예상 포인트**: 3

---

### Epic 2: 프로젝트 보드

#### US-06
**As a** 솔로 개발자,
**I want to** 여러 프로젝트를 생성하고 전환하며 관리하고 싶다,
**So that** 동시에 진행 중인 복수의 프로젝트를 하나의 툴에서 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 새 프로젝트를 이름 입력으로 생성할 수 있다
- [ ] 프로젝트 목록에서 원하는 프로젝트로 전환할 수 있다
- [ ] 프로젝트를 삭제할 수 있으며, 삭제 전 확인 다이얼로그가 표시된다
- [ ] 현재 선택된 프로젝트가 시각적으로 구분된다
- [ ] 각 프로젝트의 데이터는 독립적으로 localStorage에 저장된다

**우선순위**: Must Have
**예상 포인트**: 5

---

#### US-07
**As a** 솔로 개발자,
**I want to** Phase별로 작업 카드를 추가하고 완료/역할 배정/메모를 관리하고 싶다,
**So that** 어떤 작업이 어느 단계에 있고 누가 담당하는지 한눈에 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 각 Phase 컬럼에 작업 카드를 추가할 수 있다
- [ ] 카드에 완료 체크, 담당자 역할 배정이 가능하다
- [ ] 카드에 메모(노트)를 작성할 수 있다
- [ ] 카드에 댓글을 추가할 수 있다
- [ ] 카드 수정/삭제가 가능하다

**우선순위**: Must Have
**예상 포인트**: 5

---

#### US-08
**As a** 솔로 개발자,
**I want to** 프로젝트 데이터를 JSON 파일로 내보내고 불러오고 싶다,
**So that** 데이터를 백업하거나 다른 기기에서 동일한 상태로 작업을 이어갈 수 있다.

**Acceptance Criteria:**
- [ ] "내보내기" 버튼 클릭 시 현재 프로젝트 데이터가 `.json` 파일로 다운로드된다
- [ ] "불러오기" 버튼으로 JSON 파일을 선택하면 데이터가 복원된다
- [ ] 잘못된 형식의 JSON 파일 불러오기 시 오류 메시지가 표시된다
- [ ] 불러오기 전 현재 데이터 덮어쓰기 확인 다이얼로그가 표시된다

**우선순위**: Must Have
**예상 포인트**: 3

### Epic 3: 랜딩 & 진입점

#### US-09
**As a** 솔로 개발자,
**I want to** 프로젝트 소개 페이지에서 툴의 목적과 사용 방법을 빠르게 파악하고 싶다,
**So that** 처음 방문해도 project-board와 workflow 도구 중 무엇을 먼저 열어야 할지 알 수 있다.

**Acceptance Criteria:**

- [ ] `docs/index.html`(프로젝트 보드)과 `docs/workflow.html`(워크플로우)로 이동하는 CTA 버튼이 있다
- [ ] 기술 스택 및 주요 기능이 간략히 소개된다

**우선순위**: Must Have
**예상 포인트**: 2

### Epic 4: CLI 워크플로우 관리

#### US-10
**As a** CLI 사용자,
**I want to** 터미널에서 현재 워크플로우 진행 상태와 다음 할 일을 확인하고 싶다,
**So that** 브라우저를 열지 않고도 지금 무엇을 해야 하는지 바로 알 수 있다.

**Acceptance Criteria:**
- [ ] `node kit/cli.mjs status` 실행 시 전체 Phase/Step 완료 현황이 출력된다
- [ ] `node kit/cli.mjs next` 실행 시 다음으로 해야 할 Step과 해당 프롬프트 힌트가 출력된다
- [ ] 모든 Step이 완료된 경우 완료 메시지가 출력된다

**우선순위**: Must Have
**예상 포인트**: 2

---

#### US-11
**As a** CLI 사용자,
**I want to** 특정 Step을 완료 처리하거나 취소하고 싶다,
**So that** 브라우저 UI 없이도 워크플로우 상태를 정확하게 관리할 수 있다.

**Acceptance Criteria:**
- [ ] `node kit/cli.mjs complete <step-id>` 실행 시 해당 Step이 완료로 마킹된다
- [ ] `node kit/cli.mjs uncomplete <step-id>` 실행 시 완료 상태가 취소된다
- [ ] 존재하지 않는 step-id 입력 시 에러 메시지와 함께 사용 가능한 ID 목록이 표시된다
- [ ] 완료 처리 후 `state.json`과 `status.md`가 자동 갱신된다

**우선순위**: Must Have
**예상 포인트**: 2

---

### Epic 5: UX 개선 (Should Have)

#### US-12
**As a** 솔로 개발자,
**I want to** 다크 모드와 라이트 모드를 전환할 수 있고 싶다,
**So that** 야간 작업 시 눈의 피로를 줄일 수 있다.

**Acceptance Criteria:**
- [ ] 테마 전환 토글 버튼이 UI 상단에 표시된다
- [ ] 전환 시 CSS Custom Properties가 즉시 교체되어 전체 UI에 적용된다
- [ ] 선택한 테마가 localStorage에 저장되어 다음 방문 시 유지된다
- [ ] 시스템 다크 모드(`prefers-color-scheme`) 설정을 기본값으로 따른다

**우선순위**: Should Have
**예상 포인트**: 3

---

#### US-13
**As a** 솔로 개발자,
**I want to** 360px 이상의 모바일 화면에서도 툴을 불편 없이 사용하고 싶다,
**So that** 이동 중이나 태블릿에서도 워크플로우를 확인하고 작업을 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 360px ~ 1440px 전 구간에서 레이아웃이 깨지지 않는다
- [ ] 모바일에서 가로 스크롤 없이 모든 콘텐츠가 접근 가능하다
- [ ] 터치 타겟 크기가 44px 이상이다 (WCAG 2.5.5)
- [ ] 칸반 보드는 모바일에서 수직 스택 또는 가로 스크롤로 전환된다

**우선순위**: Should Have
**예상 포인트**: 5

---

#### US-14
**As a** 솔로 개발자,
**I want to** 작업 카드를 드래그 앤 드롭으로 다른 Phase 컬럼으로 이동하고 싶다,
**So that** 작업 상태 변경 시 직관적으로 카드를 옮길 수 있다.

**Acceptance Criteria:**
- [ ] 카드를 드래그하면 이동 가능한 컬럼이 시각적으로 강조된다
- [ ] 드롭 시 카드가 해당 Phase로 이동하고 데이터가 즉시 저장된다
- [ ] 드래그 중 원래 위치에 플레이스홀더가 표시된다
- [ ] 터치 디바이스에서도 드래그 앤 드롭이 동작한다

**우선순위**: Should Have
**예상 포인트**: 5

---

#### US-15
**As a** 솔로 개발자,
**I want to** 역할/상태/라벨 기준으로 작업 카드를 필터링하고 싶다,
**So that** 특정 담당자나 상태의 작업만 빠르게 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 역할(담당자), 완료 상태, 라벨 기준 필터 UI가 제공된다
- [ ] 필터 적용 시 조건에 맞지 않는 카드가 숨겨진다
- [ ] 복수 조건 필터링이 가능하다 (AND 조건)
- [ ] 필터 초기화 버튼으로 전체 카드를 다시 표시할 수 있다

**우선순위**: Should Have
**예상 포인트**: 3

---

#### US-16
**As a** 솔로 개발자,
**I want to** 주요 액션(복사 완료, 저장 완료 등) 시 토스트 알림을 받고 싶다,
**So that** 작업이 정상 처리되었는지 화면 이동 없이 즉시 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 프롬프트 복사, 데이터 저장, JSON 내보내기 등 주요 액션 후 토스트 메시지가 표시된다
- [ ] 토스트는 2~3초 후 자동으로 사라진다
- [ ] 에러 발생 시 다른 색상(경고/에러)으로 구분된다
- [ ] 동시에 여러 토스트가 겹쳐 표시되지 않는다

**우선순위**: Should Have
**예상 포인트**: 2
